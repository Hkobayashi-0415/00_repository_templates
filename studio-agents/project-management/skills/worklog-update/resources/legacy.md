# Worklog Update
## Purpose
作業ログを更新し、進捗状況を可視化する。

## Inputs
- 前回のワークログ
- 完了したタスク
- 発生した課題・ブロッカー
- 次のアクション

## Output Format
- 更新されたワークログエントリ
- 進捗サマリー
- 課題・リスク一覧

## Steps
1. 前回のログを確認する
2. 完了タスクを記録する
3. 進捗率を更新する
4. 発生した課題を記録する
5. 次のアクションを明記する
6. 関連ドキュメントをリンクする

## Checks (DoD)
- [ ] 日付と担当者が記載されている
- [ ] 完了タスクが具体的に記録されている
- [ ] ブロッカーがあれば明記されている
- [ ] 次のアクションが明確
- [ ] 関連チケット/PRがリンクされている

## Example Prompt
「今日の作業内容をワークログに記録してください。認証機能のPRをマージしました」
