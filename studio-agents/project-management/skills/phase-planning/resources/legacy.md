# Phase Planning
## Purpose
プロジェクトのフェーズを計画し、マイルストーンとタスクを定義する。

## Inputs
- プロジェクト概要・ゴール
- 利用可能なリソース
- 期限・制約条件
- ステークホルダー要件

## Output Format
- フェーズ定義書
- マイルストーン一覧
- タスクブレークダウン
- リスク一覧

## Steps
1. プロジェクトゴールを明確化する
2. 主要フェーズを定義する
3. 各フェーズのマイルストーンを設定する
4. タスクを洗い出し、見積もる
5. 依存関係を整理する
6. リスクを特定し、対策を検討する
7. スケジュールを作成する

## Checks (DoD)
- [ ] 全フェーズが定義されている
- [ ] マイルストーンに具体的な完了条件がある
- [ ] タスクは実行可能な粒度に分解されている
- [ ] リスクが特定され、対策がある
- [ ] ステークホルダーの合意を得ている

## Example Prompt
「新機能リリースに向けて、4週間のフェーズ計画を作成してください」
