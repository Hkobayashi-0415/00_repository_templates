# Implementation
## Purpose
設計に基づいて機能を実装し、動作するコードを作成する。

## Inputs
- 機能仕様・要件定義
- 設計ドキュメント（API設計、DB設計など）
- 技術スタック・フレームワーク情報
- 関連する既存コード

## Output Format
- 実装コード
- 単体テスト
- 実装ノート（設計判断、注意点など）

## Steps
1. 要件と設計を理解する
2. 実装計画を立てる
3. コードを段階的に実装する
4. 単体テストを作成する
5. ローカルで動作確認する
6. コードをクリーンアップする
7. ドキュメント・コメントを追加する

## Checks (DoD)
- [ ] 仕様通りに動作する
- [ ] 単体テストがパスする
- [ ] コーディング規約に準拠
- [ ] エラーハンドリングが適切
- [ ] 必要なログ出力がある
