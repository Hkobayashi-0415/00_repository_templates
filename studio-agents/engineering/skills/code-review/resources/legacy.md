# Code Review
## Purpose
コードの品質、保守性、セキュリティを確保するためのレビューを実施する。

## Inputs
- レビュー対象のプルリクエストまたはコード差分
- コーディング規約・ガイドライン
- 関連するコンテキスト（機能要件、設計書など）

## Output Format
- レビューコメント一覧
- 修正必須項目（Must）と推奨項目（Should）の分類
- 総合評価とApprove/Request Changes判定

## Steps
1. 変更の目的と影響範囲を把握する
2. コーディング規約への準拠を確認する
3. ロジックの正確性とエッジケースを検証する
4. セキュリティ上の問題がないか確認する
5. パフォーマンスへの影響を評価する
6. テストの十分性を確認する
7. コメントとフィードバックを整理する

## Checks (DoD)
- [ ] すべての変更ファイルをレビュー済み
- [ ] 重大な問題は指摘済み
- [ ] フィードバックは具体的かつ建設的
- [ ] Approve/Request Changesを決定

## Example Prompt
「このPRをレビューして、コード品質とセキュリティの観点から問題点を指摘してください」
