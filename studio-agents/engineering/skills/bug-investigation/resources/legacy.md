# Bug Investigation
## Purpose
報告されたバグの原因を特定し、修正方針を明確にする。

## Inputs
- バグレポート（再現手順、期待動作、実際の動作）
- 関連するログ・エラーメッセージ
- 影響を受けるコードベース
- 環境情報

## Output Format
- 根本原因分析（RCA）レポート
- 修正方針と影響範囲
- 再発防止策の提案

## Steps
1. バグを再現する
2. 関連するログとエラーを収集する
3. 問題箇所を特定する
4. 根本原因を分析する
5. 修正方針を策定する
6. 影響範囲を評価する
7. 調査結果をドキュメント化する

## Checks (DoD)
- [ ] バグが再現できる（または再現条件を特定）
- [ ] 根本原因が特定されている
- [ ] 修正方針が明確
- [ ] 影響範囲が把握されている
- [ ] 調査結果がドキュメント化されている

## Example Prompt
「ログイン時に500エラーが発生する問題を調査してください。エラーログは logs/error-2024-01.log にあります」
