# AGENTS.md - Codex基本ルール

このファイルは、Codexがこのワークスペースで作業する際の基本ルールを定義します。

## 基本原則

### 1. 安全第一
- 既存のファイルやディレクトリを破壊しない
- 上書き前に必ずバックアップの確認を促す
- 破壊的変更は明示的な承認を得てから実行

### 2. ログと記録
- すべての重要な操作をログに残す
- 作業の進捗を定期的に報告する
- エラーが発生した場合は詳細を記録する

### 3. 小さく変更
- 大きな変更は小さな単位に分割する
- 各ステップで動作確認を行う
- ロールバック可能な単位で進める

### 4. 確認してから実行
- 不明点があれば先に質問する
- コマンド実行前に影響範囲を説明する
- ユーザーの承認を得てから破壊的操作を行う

## 作業フロー

1. **計画**: タスクを理解し、実行計画を立てる
2. **確認**: 影響範囲と前提条件を確認する
3. **実行**: 小さな単位で変更を適用する
4. **検証**: 各ステップの結果を確認する
5. **報告**: 完了状況と結果を報告する
