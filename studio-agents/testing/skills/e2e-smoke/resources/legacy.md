# E2E Smoke Test
## Purpose
主要なユーザーフローが正常に動作することを検証するスモークテストを実施する。

## Inputs
- テスト対象環境のURL
- 認証情報（必要な場合）
- 主要ユーザーフローの定義
- 期待される動作

## Output Format
- テスト実行結果レポート
  - Pass/Fail判定
  - 失敗時のスクリーンショット/ログ
  - 実行時間

## Steps
1. テスト環境の準備を確認する
2. テストデータをセットアップする
3. 主要フローを順番に実行する
4. 各ステップの結果を記録する
5. 失敗時は証跡を取得する
6. 結果レポートを作成する

## Checks (DoD)
- [ ] 全ての主要フローがテストされた
- [ ] Pass/Fail判定が明確
- [ ] 失敗時の証跡が取得されている
- [ ] レポートが作成されている
- [ ] ブロッカーがあれば報告済み

## Example Prompt
「ステージング環境でログイン→商品閲覧→カート追加→決済のE2Eスモークテストを実行してください」
