# Test Plan
## Purpose
テスト戦略と実行計画を策定し、品質保証の方針を明確にする。

## Inputs
- テスト対象の機能仕様
- 受入基準
- リスク評価
- 利用可能なテストリソース

## Output Format
- テスト計画書
  - テスト範囲
  - テストケース一覧
  - スケジュール
  - 合否基準

## Steps
1. テスト対象と範囲を定義する
2. テスト種別を決定する（単体、結合、E2E等）
3. テストケースを作成する
4. 優先順位を設定する
5. スケジュールを策定する
6. 合否基準を定義する
7. レビューと承認を得る

## Checks (DoD)
- [ ] テスト範囲が明確
- [ ] テストケースが網羅的
- [ ] 優先順位が設定されている
- [ ] スケジュールが現実的
- [ ] 合否基準が定義されている

## Example Prompt
「新しい決済機能のテスト計画を作成してください。リリースは2週間後です」
