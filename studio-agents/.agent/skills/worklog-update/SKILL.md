# Worklog Update

## Purpose
- 作業ログを更新し、進捗状況を可視化する

## When to Use
- タスク完了時
- 日次作業終了時
- ブロッカー発生時

## Inputs
- 前回のワークログ
- 完了タスク
- 発生した課題

## Output Format
- 更新されたワークログエントリ
- 進捗サマリー
- 次のアクション

## Steps
1. 前回のログを確認
2. 完了タスクを記録
3. 進捗率を更新
4. 課題を記録
5. 次のアクションを明記

## Checks (DoD)
- [ ] 日付と担当者が記載
- [ ] 完了タスクが記録
- [ ] 次のアクションが明確

- resources/ 配下を参照

## Example Prompt
「今日の作業内容をワークログに記録してください」
